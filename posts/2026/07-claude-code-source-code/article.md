# Anthropic випадково виклали код Claude Code

![Claude Code banner](https://raw.githubusercontent.com/dolgachio/open-source-ua/refs/heads/main/posts/2026/07-claude-code-source-code/clawd-hero.jpeg)

Всім привіт, мене звати Станіслав Долгачов [(GitHub: dolgachio)](https://github.com/dolgachio). Я займаюся фронтендом у компанії EPAM і у вільний час цікавлюся опен-сорсом та веду канал [🇺🇦 OpenSourceUA](https://t.me/opensourceua) в Телеграмі.

---

31 березня разом із публікацією нової версії `Claude Code` в `Anthropic` залили source map, тим самим зливши великий шматок вихідного коду `Claude Code`. Минулого року те саме зробили [Apple з веб-версією App Store](https://dou.ua/forums/topic/56417/), тепер Anthropic туди ж. Але цього разу ситуація інша:

**По-перше**, особливого толку в коді App Store я взагалі не бачу, ну лише те, що вони там використовують `Svelte`, а не `React` (дякую Богу за це). А от код `Claude Code` — інша справа, бо багато людей його вважають еталоном агентної розробки.

**По-друге**, доступ до коду. В історії з `Apple`, як тільки злитий код хтось викладав на `GitHub`, його майже одразу видаляли, бо авторське право: код не стає опен-сорс кодом, якщо його хтось викладає в інтернет. Усі права досі **належать компанії, яка його створила**.

**Але цього разу є магічний трюк: швиденько переписати вихідний код за допомогою ШІ іншою мовою програмування**

Пан [instructkr](https://github.com/instructkr) не просто виклав код на `GitHub`, але за допомогою ШІ-агентів швиденько портував його з `TypeScript` на `Python` і виклав результат: [instructkr/claw-code](https://github.com/instructkr/claw-code). Як каже цей сміливий пан, авторське право цього разу не працює, бо це вже не той код. Поки я писав цей пост, вони його вже портували на `Rust` 🫠. Сподіваюсь, юристи `Anthropic` теж використовують ШІ для судових позовів, бо завтра воно вже буде бігать на `PHP`.
